Лаба заключается в создании программы по решению задач выбора из нескольких альтернатив методом МАИ и её модифицированным вариантом ММАИ. 

Задача по программированию решается путем преобразования в код текста из методички, совмещая теорию с примерами (и проверяя на данных из примеров). Однако там иногда сложно понять, что имеется в виду, к примеру иногда формулу без контекста предыдущего шага не понять, а есть и опечатка, которая сбивает с толку (на картинке). Причем в пособии эта опечатка выделена (а нам скинули без исправления). Из-за этого можно встрять на некоторое количество времени.

Основные сущности алгоритма - альтернативы (варианты, между которыми надо сделать итоговый выбор) и критерии (собственно критерии, по которым оцениваются альтернативы). 

У критериев может быть различная важность, выраженная в коэффициентах.

В двух словах алгоритм (хотя может это лишнее)
Для каждого из критериев строится матрица парных сравнений (МПС), где элемент на позиции i,j = отношение оценок по критерию альтернатив i и j. 
Далее эта матрица нормализуется по определенному алгоритму, затем строится оценка по критерию для каждой из альтернатив. По оценкам всех критериев строится единая оценка (оценки складываются, нормализуются), на основе этой оценки делается итоговый выбор.

Сам алгоритм метода МАИ я в методичке не нашел, возможно я просто не смог расшифровать матан) нашел объяснение на пальцах в презентации ВШЭ: https://github.com/maxi7665/KM/blob/master/MPUR_MAI.pdf

Модифицированный алгоритм описан в методичке подробнее, заключается он в том, что в ходе нескольких шагов строится МПС, элементом которого является не отношение, а нормализованный вектор, который учитывает сравнение одной альтернативы с другой по всем критериям. Это позволяет, устранить, как сказано в методичке, "противоречие, связанное с эффектом единичной нормировки, приводящей к тому, что предпочтения, выявленные на всем множестве альтернатив (критериев), могут не совпадать с «частными» предпочтениями на
подмножестве альтернатив (критериев)."  Это на второй страничке методы, в самом низу. Там и пример подобного противоречия, когда при повторном расчете порядок относительно двух альтернатив может измениться (при добавлении новой альтернативы). 
При программировании пару матриц надо построить после получения МПС (по каждому критерию) по алгоритму МАИ. По методе понять алгоритм в целом можно (по примеру в связке с теорией)

Далее  необходимо выполнить задачу по варианту,  из задачи необходимо извлечь альтернативы, критерии, веса критериев, затем запустить необходимые расчеты. Отчет по наитию сделал, в репозитории он есть (docx, pdf), как я представил результаты можно увидеть там. Посмотрим что скажет на такое препод.
